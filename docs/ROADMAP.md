# Loom Roadmap

公开维护 Loom 的长期建设方向。该文档不绑定固定时间线，采用持续演进模式，随产品与社区反馈动态调整。

## 01. 北极星（North Star）

- 将 AI 对话沉淀为可持续演进、可治理、可协作的系统记忆。
- 在不牺牲可读性与可控性的前提下，实现自动化与规模化。
- 保持 Markdown + Git 作为开放、透明、可审计的记忆载体。

## 02. 路线图使用原则

- 不以固定周期承诺交付，改为按“能力成熟度”推进。
- 优先关注长期方向，不在 roadmap 中绑定具体技术实现细节。
- 每次方向调整需有指标或用户反馈依据（见 `docs/METRICS.md`）。

## 03. 长期建设主线（Strategic Pillars）

### P1. 统一记忆操作内核

目标：同一能力只实现一次，对外通过不同入口复用。

- 统一应用服务与用例语义
- 保证 CLI、MCP 以及未来入口的一致行为
- 让系统演进从“入口驱动”转向“能力驱动”

### P2. 记忆语义骨架与知识图谱

目标：让记忆从“文档集合”演进为“可推理的知识网络”。

- 强化技术图谱与业务图谱骨架
- 持续提升知识条目间的可追踪关系
- 支持影响分析与跨主题检索

### P3. 质量治理与发布门禁

目标：把知识质量变成可治理的工程能力，而非人工自觉。

- 形成可配置的质量策略体系
- 建立写入、检索、发布前的门禁机制
- 让治理标准可复用、可审查、可回归

### P4. 数据反馈闭环与决策驱动

目标：从“有日志”升级为“可量化、可复盘、可决策”。

- 建立统一指标事实源
- 建立事件到指标映射
- 建立指标到决策的固定复盘流程
- 建立“原始对话数据层”（可配置全量记录），用于评估“真实对话输入 -> 最终知识沉淀”的有效性偏差

### P5. 生态与协作规模化

目标：让 Loom 形成可持续社区协作机制。

- 降低贡献门槛与接入成本
- 标准化 PR 验证流程与文档更新要求
- 逐步形成“社区共建路线图”的产品治理模式

## 04. 能力成熟度视图（非时间线）

### Level 1 — 可用

- 可写、可读、可检索、可同步
- 基础自动化可运行

### Level 2 — 可治理

- 质量规则与图谱规则稳定执行
- 回归机制可复现

### Level 3 — 可决策

- 指标可持续采集
- 复盘驱动下一步优先级

### Level 4 — 可规模化

- 多入口一致
- 多贡献者高质量协作
- 方向演进可持续

## 05. 指标对齐

- 指标定义与周度模板见：`docs/METRICS.md`
- 每个 roadmap 变更建议标注影响指标（M1/M2/M3）
- 方向优先级调整应有指标变化或明确用户反馈作为依据

## 06. 协作与提案机制

- 想法收集：`docs/BRAINSTORM.md`
- 任务拆解：`docs/IMPLEMENTATION_PLAN.md`
- 路线更新：直接 PR 修改本文件

每个 roadmap PR 建议包含：

- 背景问题
- 方向收益
- 影响范围
- 关键风险
- 指标影响假设（对应 `docs/METRICS.md`）
